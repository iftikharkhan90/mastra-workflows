## ğŸ“– Overview

This Mastra application implements a **multi-agent routing system** with **workflow-based processing**. User queries are analyzed, routed to the appropriate workflow, processed through multiple steps, and returned with AI-generated responses.

### ğŸ¯ Key Features:
- **3 AI Agents**: Router, Vehicle Expert, Booking Expert
- **2 Workflows**: Vehicle Recommendations, Booking Assistance
- **4 Tools**: Workflow executors and direct agent calls
- **Detailed Logging**: Complete execution flow visibility

### Pattern 1: Workflow Registration (index.ts)

```typescript

  workflows: {

    'vehicle-workflow': vehicleWorkflow,   // âœ… Correct
    'booking-workflow': bookingWorkflow,   // âœ… Correct
    
    // âŒ WRONG: vehicleWorkflow (would register as 'vehicleWorkflow')
  },
  agents: { ... },

```

### Pattern 2: Workflow Execution (router-tools.ts)

```typescript
async function executeWorkflow(workflowId, input) {
  // 1. Get workflow
  const workflow = mastra.getWorkflow(workflowId);
  
  // 2. Create run (NEW API - async)
  const run = await workflow.createRunAsync({ runId });
  
  // 3. Start execution
  const result = await run.start({ inputData: input });
  
  // 4. Check result
  if (result.status === 'success') {
    return result.result;
  }
}
```

### Pattern 3: Workflow Step Definition

```typescript
const myStep = createStep({
  id: 'step-id',
  description: 'What this step does',
  inputSchema: z.object({ ... }),   // Zod schema for input
  outputSchema: z.object({ ... }),  // Zod schema for output
  execute: async ({ inputData, mastra }) => {
    // inputData = output from previous step (or workflow input)
    // mastra = access to agents and other resources
    
    const agent = mastra.getAgent('agentName');
    const response = await agent.generate([...]);
    
    return { ... };  // Must match outputSchema
  },
});
```

### Pattern 4: Workflow Chaining

```typescript
const workflow = createWorkflow({
  id: 'my-workflow',
  inputSchema: z.object({ query: z.string() }),
  outputSchema: finalOutputSchema,
})
  .then(step1)  // Step 1 output â†’ Step 2 input
  .then(step2)  // Step 2 output â†’ Step 3 input
  .then(step3); // Step 3 output = Final output

workflow.commit();  // Finalize the workflow
```

### Bug Fix #1: Workflow Registration Key Mismatch

**Problem:**
```typescript
// Workflows registered with variable names as keys
workflows: { 
  vehicleWorkflow,    // Key = "vehicleWorkflow"
  bookingWorkflow,    // Key = "bookingWorkflow"
}

// Tool looking for different keys
mastra.getWorkflow('booking-workflow')  // âŒ Not found!
```

**Solution:**
```typescript
// Register with explicit string keys matching workflow IDs
workflows: { 
  'vehicle-workflow': vehicleWorkflow,   // Key = "vehicle-workflow" âœ…
  'booking-workflow': bookingWorkflow,   // Key = "booking-workflow" âœ…
}
```

**File Changed:** `src/mastra/index.ts`

**Problem:**

## ğŸ”§ Troubleshooting

### Error: "Workflow with ID xxx not found"

**Cause:** Workflow key in registration doesn't match the ID used in `getWorkflow()`

**Fix:** Ensure workflow registration uses string keys:
```typescript
workflows: { 
  'workflow-id': workflowVariable,
}
```

### Error: "createRun() has been deprecated"

**Cause:** Using old synchronous API

**Fix:** Use the new async API:
```typescript
// Old (deprecated)
const run = workflow.createRun({ runId });

// New (correct)
const run = await workflow.createRunAsync({ runId });
```

### Error: "Mastra instance not initialized"

**Cause:** `setMastraInstance()` not called before tool execution

**Fix:** Ensure `setMastraInstance(mastra)` is called after Mastra instance creation in `index.ts`

### Error: "Agent not found"

**Cause:** Agent not registered with Mastra or wrong agent name

**Fix:** 
1. Verify agent is in `agents: { ... }` in Mastra config
2. Use correct agent name in `mastra.getAgent('agentName')`
